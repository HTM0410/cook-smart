# Database Schema - CookSmart Food Suggest Application

## Tổng quan
Database `cooksmart` được thiết kế để hỗ trợ ứng dụng gợi ý món ăn thông minh với các tính năng:
- Quản lý người dùng và xác thực
- Quản lý công thức nấu ăn chi tiết
- Hệ thống nguyên liệu và phân loại
- Đánh giá và yêu thích công thức
- Hệ thống bình luận và tương tác
- Quản lý admin và phê duyệt

## Cấu trúc Database

### 1. Bảng `users` - Quản lý người dùng
```sql
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    full_name VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    avatar VARCHAR(255) NULL,
    status ENUM('active', 'banned') DEFAULT 'active',
    role ENUM('user', 'admin') DEFAULT 'user',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `full_name`: Họ tên đầy đủ của người dùng (tối đa 100 ký tự)
- `email`: Email duy nhất để đăng nhập (tối đa 100 ký tự)
- `password`: Mật khẩu đã được hash bằng bcrypt (12 rounds)
- `avatar`: URL ảnh đại diện (tùy chọn)
- `status`: Trạng thái tài khoản (active/banned)
- `role`: Vai trò người dùng (user/admin)
- `created_at`, `updated_at`: Timestamp tự động

**Indexes:**
- UNIQUE KEY trên `email`
- INDEX trên `status`, `role`

### 2. Bảng `recipes` - Công thức nấu ăn
```sql
CREATE TABLE recipes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    recipe_name VARCHAR(150) NOT NULL,
    description TEXT NULL,
    image_url VARCHAR(255) NULL,
    prep_time INT NOT NULL,
    cook_time INT NOT NULL,
    servings INT NOT NULL,
    difficulty ENUM('easy', 'medium', 'hard') NOT NULL,
    created_by INT NOT NULL,
    status ENUM('visible', 'hidden') DEFAULT 'visible',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `recipe_name`: Tên công thức (tối đa 150 ký tự)
- `description`: Mô tả chi tiết công thức
- `image_url`: URL ảnh minh họa
- `prep_time`: Thời gian chuẩn bị (phút)
- `cook_time`: Thời gian nấu (phút)
- `servings`: Số khẩu phần
- `difficulty`: Độ khó (easy/medium/hard)
- `created_by`: ID người tạo (FK → users.id)
- `status`: Trạng thái hiển thị (visible/hidden)

**Indexes:**
- INDEX trên `recipe_name`, `difficulty`, `status`, `created_by`

### 3. Bảng `ingredients` - Nguyên liệu
```sql
CREATE TABLE ingredients (
    id INT AUTO_INCREMENT PRIMARY KEY,
    ingredient_name VARCHAR(100) NOT NULL UNIQUE,
    category_id INT NOT NULL,
    description TEXT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `ingredient_name`: Tên nguyên liệu (duy nhất, tối đa 100 ký tự)
- `category_id`: ID danh mục (FK → ingredient_categories.id)
- `description`: Mô tả nguyên liệu

**Indexes:**
- UNIQUE KEY trên `ingredient_name`
- INDEX trên `category_id`

### 4. Bảng `ingredient_categories` - Danh mục nguyên liệu
```sql
CREATE TABLE ingredient_categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_name VARCHAR(100) NOT NULL UNIQUE,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `category_name`: Tên danh mục (duy nhất, tối đa 100 ký tự)

**Indexes:**
- UNIQUE KEY trên `category_name`

### 5. Bảng `recipe_ingredients` - Nguyên liệu của công thức
```sql
CREATE TABLE recipe_ingredients (
    id INT AUTO_INCREMENT PRIMARY KEY,
    recipe_id INT NOT NULL,
    ingredient_id INT NOT NULL,
    quantity VARCHAR(50) NOT NULL,
    unit VARCHAR(20) NULL,
    notes TEXT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `recipe_id`: ID công thức (FK → recipes.id)
- `ingredient_id`: ID nguyên liệu (FK → ingredients.id)
- `quantity`: Số lượng (có thể là số hoặc text như "1/2", "một ít")
- `unit`: Đơn vị đo (g, kg, ml, thìa, cốc, v.v.)
- `notes`: Ghi chú thêm

**Indexes:**
- INDEX trên `recipe_id`, `ingredient_id`
- UNIQUE KEY trên `(recipe_id, ingredient_id)` để tránh trùng lặp

### 6. Bảng `recipe_steps` - Các bước thực hiện
```sql
CREATE TABLE recipe_steps (
    id INT AUTO_INCREMENT PRIMARY KEY,
    recipe_id INT NOT NULL,
    step_number INT NOT NULL,
    instruction TEXT NOT NULL,
    image_url VARCHAR(255) NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `recipe_id`: ID công thức (FK → recipes.id)
- `step_number`: Số thứ tự bước (1, 2, 3, ...)
- `instruction`: Hướng dẫn chi tiết bước đó
- `image_url`: URL ảnh minh họa bước (tùy chọn)

**Indexes:**
- INDEX trên `recipe_id`
- UNIQUE KEY trên `(recipe_id, step_number)` để đảm bảo thứ tự

### 7. Bảng `user_favorites` - Món yêu thích
```sql
CREATE TABLE user_favorites (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    recipe_id INT NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `user_id`: ID người dùng (FK → users.id)
- `recipe_id`: ID công thức (FK → recipes.id)

**Indexes:**
- INDEX trên `user_id`, `recipe_id`
- UNIQUE KEY trên `(user_id, recipe_id)` để tránh trùng lặp

### 8. Bảng `recipe_reviews` - Đánh giá công thức
```sql
CREATE TABLE recipe_reviews (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    recipe_id INT NOT NULL,
    rating INT NOT NULL,
    comment TEXT NULL,
    is_active TINYINT(1) DEFAULT 1,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `user_id`: ID người dùng (FK → users.id)
- `recipe_id`: ID công thức (FK → recipes.id)
- `rating`: Điểm đánh giá (1-5 sao)
- `comment`: Bình luận đánh giá
- `is_active`: Trạng thái hoạt động (1: hiển thị, 0: ẩn)

**Indexes:**
- INDEX trên `user_id`, `recipe_id`, `rating`
- UNIQUE KEY trên `(user_id, recipe_id)` để mỗi user chỉ đánh giá 1 lần

### 9. Bảng `comments` - Hệ thống bình luận
```sql
CREATE TABLE comments (
    id INT AUTO_INCREMENT PRIMARY KEY,
    recipe_id INT NOT NULL,
    user_id INT NOT NULL,
    parent_id INT NULL,
    content TEXT NOT NULL,
    is_edited TINYINT(1) DEFAULT 0,
    edited_at DATETIME NULL,
    is_deleted TINYINT(1) DEFAULT 0,
    deleted_at DATETIME NULL,
    like_count INT DEFAULT 0,
    reply_count INT DEFAULT 0,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `recipe_id`: ID công thức (FK → recipes.id)
- `user_id`: ID người dùng (FK → users.id)
- `parent_id`: ID bình luận cha (để hỗ trợ reply, NULL nếu là bình luận gốc)
- `content`: Nội dung bình luận
- `is_edited`: Đã chỉnh sửa chưa
- `edited_at`: Thời gian chỉnh sửa
- `is_deleted`: Đã xóa chưa (soft delete)
- `deleted_at`: Thời gian xóa
- `like_count`: Số lượt thích
- `reply_count`: Số reply

**Indexes:**
- INDEX trên `recipe_id`, `user_id`, `parent_id`

### 10. Bảng `admins` - Quản trị viên
```sql
CREATE TABLE admins (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    role ENUM('superadmin', 'moderator') DEFAULT 'moderator',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `username`: Tên đăng nhập admin (duy nhất)
- `password`: Mật khẩu đã hash
- `role`: Vai trò admin (superadmin/moderator)

**Indexes:**
- UNIQUE KEY trên `username`

### 11. Bảng `pending_ingredients` - Nguyên liệu chờ phê duyệt
```sql
CREATE TABLE pending_ingredients (
    id INT AUTO_INCREMENT PRIMARY KEY,
    ingredient_name VARCHAR(100) NOT NULL,
    submitted_by INT NULL,
    status ENUM('pending', 'approved', 'rejected') DEFAULT 'pending',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    category_id INT NOT NULL,
    description TEXT NULL,
    reviewed_by INT NULL,
    reviewed_at DATETIME NULL,
    rejection_reason TEXT NULL
);
```

**Mô tả:**
- `id`: Khóa chính, tự động tăng
- `ingredient_name`: Tên nguyên liệu đề xuất
- `submitted_by`: ID người đề xuất (FK → users.id)
- `status`: Trạng thái phê duyệt (pending/approved/rejected)
- `category_id`: ID danh mục đề xuất (FK → ingredient_categories.id)
- `description`: Mô tả nguyên liệu
- `reviewed_by`: ID admin phê duyệt (FK → admins.id)
- `reviewed_at`: Thời gian phê duyệt
- `rejection_reason`: Lý do từ chối

**Indexes:**
- INDEX trên `ingredient_name`, `status`, `created_at`, `submitted_by`, `reviewed_by`

## Quan hệ giữa các bảng (Foreign Keys)

### Quan hệ chính:
1. **users** → **recipes** (1:N): `recipes.created_by` → `users.id`
2. **recipes** → **recipe_ingredients** (1:N): `recipe_ingredients.recipe_id` → `recipes.id`
3. **ingredients** → **recipe_ingredients** (1:N): `recipe_ingredients.ingredient_id` → `ingredients.id`
4. **ingredient_categories** → **ingredients** (1:N): `ingredients.category_id` → `ingredient_categories.id`
5. **recipes** → **recipe_steps** (1:N): `recipe_steps.recipe_id` → `recipes.id`
6. **users** → **user_favorites** (1:N): `user_favorites.user_id` → `users.id`
7. **recipes** → **user_favorites** (1:N): `user_favorites.recipe_id` → `recipes.id`
8. **users** → **recipe_reviews** (1:N): `recipe_reviews.user_id` → `users.id`
9. **recipes** → **recipe_reviews** (1:N): `recipe_reviews.recipe_id` → `recipes.id`
10. **recipes** → **comments** (1:N): `comments.recipe_id` → `recipes.id`
11. **users** → **comments** (1:N): `comments.user_id` → `users.id`
12. **comments** → **comments** (1:N): `comments.parent_id` → `comments.id` (self-reference)
13. **users** → **pending_ingredients** (1:N): `pending_ingredients.submitted_by` → `users.id`
14. **admins** → **pending_ingredients** (1:N): `pending_ingredients.reviewed_by` → `admins.id`
15. **ingredient_categories** → **pending_ingredients** (1:N): `pending_ingredients.category_id` → `ingredient_categories.id`

## Constraints và Ràng buộc

### Ràng buộc dữ liệu:
- **Password**: Tối thiểu 8 ký tự, phải có ít nhất 1 chữ và 1 số
- **Email**: Format email hợp lệ, duy nhất
- **Rating**: Giá trị từ 1-5
- **Status**: Chỉ chấp nhận các giá trị enum được định nghĩa
- **Timestamps**: Tự động cập nhật `updated_at` khi có thay đổi

### Ràng buộc nghiệp vụ:
- Mỗi user chỉ có thể đánh giá 1 lần cho mỗi recipe
- Mỗi user chỉ có thể yêu thích 1 lần cho mỗi recipe
- Recipe steps phải có thứ tự liên tục (1, 2, 3, ...)
- Soft delete cho comments (không xóa thật)

## Performance và Tối ưu

### Indexes được tạo:
- Primary keys trên tất cả bảng
- Unique constraints trên email, username, ingredient_name
- Composite indexes cho các truy vấn thường xuyên
- Indexes trên foreign keys
- Indexes trên các trường tìm kiếm (recipe_name, ingredient_name)

### Caching Strategy:
- Redis cache cho kết quả tìm kiếm phổ biến
- Cache cho user sessions và JWT tokens
- Cache cho recipe details và ingredients

## Backup và Maintenance

### Backup Strategy:
- Full backup hàng ngày
- Incremental backup mỗi 6 giờ
- Backup trước khi deploy

### Maintenance:
- Cleanup các bản ghi đã xóa (soft delete) định kỳ
- Optimize tables hàng tuần
- Monitor performance và slow queries
